**5.1. Порядок проверки отчёта**

1.  **Запуск кода:** Kernel → Restart & Run All. Код должен отработать без ошибок.
2.  **Проверка масштабирования:** Убедитесь, что scaler.fit() вызван только на трейне. Если студент сделал fit_transform на всем датасете до train_test_split — это грубая ошибка (утечка данных), снижение на 0.2 балла.
3.  **Проверка полноты ответов:** Пройдитесь по всем ячейкам Markdown. Ответы в стиле «потому что так лучше» без математического/логического обоснования не принимаются.
