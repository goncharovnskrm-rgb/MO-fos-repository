**3\. Данные**

В лабораторной работе №2 используются два датасета:

**3.1. Pima Indians Diabetes Dataset (задача классификации с регуляризацией)**

| **Параметр** | **Значение** |
| --- | --- |
| Источник | UCI Machine Learning Repository, загружается напрямую с GitHub через pd.read_csv(url) |
| URL | https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.csv |
| Объём | 768 объектов, 8 признаков |
| Целевая переменная | class – наличие диабета (0 = нет, 1 = есть), бинарная |
| Типы признаков | Все числовые: pregnancies, glucose, blood_pressure, skin_thickness, insulin, bmi, diabetes_pedigree, age |
| Особенности | Содержит мультиколлинеарные признаки – идеальная среда для демонстрации L1/L2 регуляризации. Сбалансирован (65% класс 0, 35% класс 1) – позволяет сфокусироваться на регуляризации, не отвлекаясь на дисбаланс. Небольшой – быстро обучается, можно многократно запускать GridSearchCV. Классический бенчмарк – студенты смогут сравнить результаты с литературой. |
| Размер на диске | ~30 КБ (не кэшируется sklearn, загружается каждый раз по URL) |

**Примечание для преподавателя:**

1.  Загрузка Diabetes занимает 1–2 секунды в зависимости от скорости интернета.
2.  Если интернет-соединение в аудитории нестабильно, преподаватель может заранее загрузить датасет на одном компьютере и сохранить его в формате .csv, а затем раздать студентам. В этом случае студентам потребуется заменить pd.read_csv(url, ...) на pd.read_csv('pima-indians-diabetes.csv', ...).
3.  В датасете присутствуют пропущенные значения (закодированные как 0 в признаках glucose, blood_pressure, skin_thickness, insulin, bmi). В данной работе они не обрабатываются специально, так как цель – демонстрация регуляризации, а не предобработки. Однако преподаватель может упомянуть это студентам как тему для обсуждения.

**3.2. Credit Card Fraud Detection Dataset (задача классификации с экстремальным дисбалансом)**

| **Параметр** | **Значение** |
| --- | --- |
| Источник | Kaggle / ULB (Université Libre de Bruxelles), загружается через sklearn.datasets.fetch_openml(data_id=1597) |
| Объём (полный) | 284 807 транзакций, 30 признаков |
| Объём (в работе) | ~20 492 объекта (субсэмплинг: все 492 мошеннических + 20 000 случайных легальных) |
| Целевая переменная | Class – тип транзакции (0 = легальная, 1 = мошенничество), бинарная |
| Типы признаков | Time – секунды между транзакциями, Amount – сумма транзакции, V1–V28 – PCA-компоненты (анонимизированы для конфиденциальности) |
| Особенности | **Экстремальный дисбаланс** (99.83% легальных, 0.17% мошенничеств в полном датасете; после субсэмплинга – ~2.4% мошенничеств) – идеально для демонстрации SMOTE, ADASYN, class weights, Focal Loss, пороговой калибровки. Реальная бизнес-задача – обнаружение мошенничества с асимметричной стоимостью ошибок (FN намного дороже FP). PCA-признаки – ортогональны, что позволяет обсудить, почему Lasso работает хуже на таких данных. Промышленный стандарт – используется в курсах и соревнованиях по всему миру. |
| Размер на диске | ~60–70 МБ (кэшируется в ~/.cache/scikit_learn_data/openml/) |

**Примечание для преподавателя:**

1.  **Первая загрузка Fraud может занять 1–3 минуты** в зависимости от скорости интернета. Рекомендуется попросить студентов запустить ячейку загрузки заранее (в начале занятия).
2.  Если интернет-соединение в аудитории нестабильно, преподаватель может заранее загрузить датасет на одном компьютере и сохранить его в формате .csv или .parquet, а затем раздать студентам. В этом случае студентам потребуется заменить fetch_openml(data_id=1597, ...) на pd.read_csv(...).
3.  **Субсэмплинг фиксирован** (np.random.seed(42)), поэтому результаты у всех студентов должны быть **идентичны** при одинаковых параметрах моделей. Это упрощает проверку.
4.  В коде ноутбука target преобразуется через pd.to_numeric(..., errors='coerce') на случай, если он загружается как строковый. Если target уже числовой, эта операция не требуется, но она безопасна и не влияет на результат.
5.  **Важно:** после субсэмплинга дисбаланс составляет ~2.4% (492 мошенничества против 20 000 легальных транзакций). Это всё ещё сильный дисбаланс, достаточный для демонстрации всех методов работы с несбалансированными данными.

**4\. Дополнительные рекомендации по организации работы**

1.  **Порядок выполнения заданий:** Рекомендуется выполнять задания последовательно (Задание 1 → Задание 2 → Задание 3 → Задание 4 → Задание 5), так как они построены по принципу «от простого к сложному» и используют результаты предыдущих шагов.
2.  **Задание 3 (Focal Loss с нуля)** – наиболее сложное. Студентам потребуется реализовать собственный класс, наследующий от BaseEstimator и ClassifierMixin. Преподаватель должен быть готов помочь с API sklearn для создания кастомных оценщиков. В коде ноутбука приведена подсказка с шаблоном класса.
3.  **Задание 5 (бонусное)** – не является обязательным для получения максимальной оценки, но рекомендуется для глубокого понимания важности стратификации. Студенты, выполнившие это задание, получают дополнительные баллы к качеству выводов.
4.  **Воспроизводимость:** Во всех случайных процессах необходимо фиксировать random_state=42. Это обеспечит идентичность результатов у всех студентов и упростит проверку.
5.  **Визуализация:** Все графики должны быть подписаны (оси, заголовки, легенда). К каждому графику должны быть текстовые выводы в формате Markdown. Поверхностные выводы («метрика стала лучше») без объяснения причин – основание для снижения оценки.
6.  **Время выполнения:** Ориентировочное время выполнения работы – 4 часа аудиторной работы + 2 часа самостоятельной работы. Если студенты не успевают в аудитории, они могут завершить работу дома.
7.  **Частые ошибки студентов:**
    - Применение SMOTE ко всему датасету до разделения на train/test (data leakage).
    - Использование обычного KFold вместо StratifiedKFold для несбалансированных данных.
    - Забыть масштабирование признаков перед LogisticRegression (градиентные методы чувствительны к масштабу).
    - Неправильное наследование в Задании 3 (забыть BaseEstimator или ClassifierMixin).
    - Путаница между параметром C (обратная сила регуляризации) и alpha (сила регуляризации).
