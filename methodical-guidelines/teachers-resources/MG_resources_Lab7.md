**4\. Дополнительные рекомендации по организации работы**

**4.1. Порядок выполнения заданий**

Рекомендуется выполнять задания последовательно (Задание 1 → Задание 4), так как:

1.  Задание 1 (Swiss Roll) формирует геометрическую интуицию: почему линейные методы не работают на нелинейных многообразиях.
2.  Задание 2 (Covertype) требует понимания разницы между Feature Selection и Feature Extraction, а также навыка интерпретации многоклассовых визуализаций.
3.  Задание 3 (Digits) развивает навык подбора гиперпараметров и понимания их геометрического смысла.
4.  Задание 4 (Индивидуальное) – самостоятельное комплексное применение всех изученных методов на новом датасете.

**4.2. t-SNE и UMAP – «узкое место» по времени**

Студенты могут попытаться запустить t-SNE на полном Covertype (581 012 объектов) или на 10 000 объектов с дефолтными параметрами. Это приведёт к зависанию ноутбука на 30–60 минут.

**Решение:**

1.  В ноутбуке t-SNE для Covertype намеренно пропущен с пояснением. Студентам рекомендуется запускать t-SNE только на подвыборках 1000–2000 объектов.
2.  Для Digits (1797 объектов) t-SNE работает за 1–3 минуты – это допустимо.
3.  Обязательно требовать использования n_jobs=-1 в RandomForestClassifier.
4.  Для UMAP рекомендовать параметры по умолчанию (n_neighbors=15, min_dist=0.1). UMAP работает в 10–50 раз быстрее t-SNE на тех же данных.
5.  Если студент работает локально на слабом ноутбуке, разрешить уменьшить n_iter для t-SNE до 500 (вместо 1000 по умолчанию).

**4.3. Воспроизводимость**

Во всех случайных процессах необходимо фиксировать random_state=42. Это обеспечит идентичность результатов (особенно важно для make_swiss_roll, PCA, TSNE, UMAP, SelectKBest, RandomForestClassifier и train_test_split).

**4.4. Визуализация**

1.  **Swiss Roll 3D + проекции (Задание 1)** – обязательный элемент. Студент должен видеть 4 графика: оригинал, PCA, KernelPCA/t-SNE, UMAP. Цвет точек – по позиции на ленте (непрерывная шкала).
2.  **Многоклассовые scatter-plot (Задания 2, 3)** – обязательный элемент. Для Covertype (7 классов) и Digits (10 классов) графики должны содержать: заголовок, подписи осей («Компонента 1», «Компонента 2»), легенду с названиями классов. Рекомендуется ncol=2 в легенде для компактности.
3.  **График explained variance ratio (Задание 2)** – обязательный элемент. Студент должен построить график кумулятивной объяснённой дисперсии PCA и указать, сколько компонент нужно для 90%/95% дисперсии.
4.  **Серия графиков с разными гиперпараметрами (Задание 3)** – обязательный элемент. Минимум 3 значения perplexity для t-SNE и 3 комбинации n_neighbors/min_dist для UMAP, расположенных в сетке subplot.
5.  **Индивидуальная визуализация (Задание 4)** – обязательный элемент. Scatter-plot с легендой, заголовком, подписями осей. Цвета классов должны быть различимы.

**4.6. Частые ошибки студентов**

| **№** | **Ошибка** | **Как проверить / исправить** |
| --- | --- | --- |
| **1** | **Отсутствие StandardScaler** перед PCA, KernelPCA, t-SNE, UMAP и SelectKBest. Студент применяет методы к сырым данным. | Проверить наличие scaler = StandardScaler(); X_scaled = scaler.fit_transform(X). Для PCA это критично: признак с большей дисперсией (например, Proline ~1000) полностью подавит признак с малой дисперсией (Hue ~1). |
| **2** | **Не установлена библиотека umap-learn.** Студент получает ModuleNotFoundError: No module named 'umap'. | Убедиться, что студент выполнил !pip install umap-learn в первой ячейке (Colab) или установил локально. Проверить: import umap; print(umap.\__version_\_). |
| **3** | **Студент пытается запустить t-SNE на полном Covertype (581 012 объектов).** Ноутбук зависает или падает по памяти. | Объяснить, что t-SNE имеет сложность O(N²) по памяти и O(N·log N) по времени (с Barnes-Hut). Для Covertype допустима подвыборка ≤ 2000 объектов. UMAP масштабируется значительно лучше. |
| **4** | **Путаница между Feature Selection и Feature Extraction.** Студент называет PCA «отбором признаков». | На защите спросить: «Что делает SelectKBest? А что делает PCA? Можно ли после PCA объяснить, какой исходный признак наиболее важен?» (Ответ: после PCA – нет, это новые синтетические признаки; после SelectKBest – да, это подмножество исходных). |
| **5** | **Непонимание параметра perplexity в t-SNE.** Студент ставит perplexity=500 для 1797 объектов или perplexity=2 для 10 000 объектов. | Объяснить: perplexity должна быть значительно меньше числа объектов (обычно 5–50). Грубо: это «эффективное число соседей». Малое значение дробит кластеры, большое – сливает их в одно пятно. |
| **6** | **Студент интерпретирует расстояния между кластерами на t-SNE проекции как значимые.** | Объяснить: t-SNE сохраняет только **локальную** структуру. Расстояние между далёкими кластерами на графике t-SNE **не несёт смысла**. UMAP в этом отношении надёжнее (сохраняет глобальную структуру). |
| **7** | **В индивидуальном задании (Задание 4) датасет имеет < 20 признаков.** | Требовать минимум 20 признаков. Подходящие источники: Kaggle (например, кредитный скоринг, телеком-отток), UCI ML Repository, генетические данные, TF-IDF текстов. |
| **8** | **PCA объясняет <30% дисперсии на Covertype, и студент считает это ошибкой.** | Объяснить: 54 признака Covertype включают 10 бинарных one-hot признаков почвенного типа. Дисперсия «размазана» по многим измерениям. Это не ошибка, а свойство данных. UMAP при этом чётко разделяет классы, потому что работает с локальной структурой, а не с глобальной дисперсией. |
| **9** | **Студент использует TSNE с методом transform() для новых данных.** | Объяснить: t-SNE **не имеет** метода transform() (только fit_transform). Это принципиальное отличие от PCA. Для проекции новых точек нужен параметрический t-SNE или UMAP (у UMAP есть transform()). |
| **10** | **Поверхностные выводы.** Студент пишет «PCA – линейный, t-SNE – нелинейный, UMAP – лучше», не объясняя геометрический смысл. | Требовать объяснения: PCA ищет оси максимальной дисперсии; t-SNE минимизирует KL-дивергенцию между распределениями расстояний в исходном и проекционном пространствах; UMAP строит fuzzy simplicial set (топологический граф) и оптимизирует его вложение. |
| **11** | **В Задании 2 (Covertype) студент не строит график зависимости Accuracy от k в SelectKBest.** | Это обязательный эксперимент (k=5, 10, 20, 30, 54). Студент должен построить график и обосновать выбор оптимального k (обычно по «локтю» или плато). |
| **12** | **Легенда на многоклассовом графике перекрывает данные.** | Рекомендовать: loc='best', ncol=2, fontsize=9, или выносить легенду за пределы графика (bbox_to_anchor=(1.05, 1)). Для 10 классов Digits использовать компактную сетку маркеров. |
