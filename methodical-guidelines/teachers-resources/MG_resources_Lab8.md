**4\. Дополнительные рекомендации по организации работы**

**4.1. Порядок выполнения заданий**

Рекомендуется выполнять задания последовательно:

1.  **Задание 1 (Baseline)** — загрузка данных, предобработка, построение простой линейной регрессии без дополнительных предикторов. Формирует базовое понимание задачи и качества данных.
2.  **Задание 2 (Feature Engineering)** — применение L1/L2-регуляризации, кросс-валидации, полиномиальных признаков, создание логических признаков (Xi ≥ 0.6·max(Xi)), оценка важности признаков, комбинирование признаков. Развивает навыки тонкой настройки моделей.
3.  **Задание 3 (AutoML)** — применение пакета auto-sklearn для автоматического подбора модели и гиперпараметров. Сравнение результатов с baseline и ручной моделью. **Ключевое задание работы.**
4.  **Задание 4 (Выводы)** — формулирование итоговых выводов по исследованию, интерпретация бизнес-смысла прогноза.

**4.2. Auto-sklearn — «узкое место» по времени и памяти**

Студенты могут столкнуться с двумя основными проблемами при работе с auto-sklearn:

1.  **Длительное время обучения.** По умолчанию auto-sklearn выполняет поиск моделей в течение 3600 секунд (1 час). Это недопустимо в рамках аудиторных занятий.
    - Решение: явно задавать параметр time_left_for_this_task (рекомендуемые значения: 120–360 секунд для учебных целей) и per_run_time_limit (30–60 секунд).
    - Пример:

&nbsp;  automl = autosklearn.regression.AutoSklearnRegressor(

&nbsp;      time_left_for_this_task=240,

&nbsp;      per_run_time_limit=60,

&nbsp;      random_state=42

&nbsp;  )

1.  **Требования к памяти.** auto-sklearn хранит в памяти множество обученных моделей, что может привести к MemoryError.
    - **Решение:** использовать параметр memory_limit (рекомендуемое значение: 2048–4096 МБ) или уменьшить размер выборки.
2.  **Предупреждения и логи.** auto-sklearn генерирует большое количество предупреждений. Рекомендуется использовать warnings.filterwarnings('ignore') для чистоты вывода.

**4.3. Воспроизводимость**

Во всех случайных процессах необходимо фиксировать random_state=42. Это обеспечит идентичность результатов (особенно важно для train_test_split, LinearRegression, Ridge, Lasso, auto-sklearn).

**4.4. Визуализация**

1.  **Гистограмма распределения итогового балла** (data3\['sumball'\].hist()) — обязательный элемент для понимания задачи.
2.  **Pairplot зависимостей** (sns.pairplot(data_3\[lasttasks\], hue='label')) — обязательный элемент для демонстрации статистической связи признаков с целевой переменной.
3.  **Scatter-график отклонений прогноза** (plt.scatter(y_test, y_predicted)) — обязательный элемент для анализа качества модели. На графике должна быть диагональная линия y=x для идеального прогноза.
4.  **Графики важности признаков** (feature_importances_) — обязательный элемент для заданий 2 и 3.

**4.5. Время выполнения**

Ориентировочное время — 4 часа аудиторной работы (задания 1–3, разбор кода) + 2 часа самостоятельной работы (задание 4 — формулирование выводов, подготовка к защите).

**Важно:** при использовании auto-sklearn с time_left_for_this_task=240 только это задание займёт ~4 минуты. Преподавателю рекомендуется планировать время с учётом этого.

**4.7. Рекомендации по проверке работ**

При проверке работ преподавателю рекомендуется обратить особое внимание на:

1.  **Качество предобработки данных** — это основа всей работы. Без корректной предобработки дальнейшие шаги бессмысленны.
2.  **Корректность разделения выборки** — train/test split должен выполняться до любой предобработки (скейлинг, полиномиальные признаки).

4.  **Интерпретация результатов** — студент должен объяснить бизнес-смысл прогноза (например, раннее выявление студентов группы риска для оказания дополнительной поддержки).
5.  **Время выполнения auto-sklearn** — если студент не ограничил время, работа может не запуститься при проверке.
